# Docker Load Map

## 📍 Phase 1: コンテナの「魔法」を解く（LinuxカーネルとOCI）
Dockerは魔法ではなく、Linuxの機能の組み合わせなんだ。ここを知ると、すべてのトラブルシューティングの解像度が跳ね上がるよ！

- 学ぶこと: namespaces (隔離)、cgroups (リソース制限)
- 学ぶこと: ストレージドライバ (OverlayFS) とコンテナの書き込みメカニズム
- 学ぶこと: OCI (Open Container Initiative) 仕様とは何か？

## 📍 Phase 2: イメージの完全解剖学（レイヤー・マニフェスト・ダイジェスト）
「イメージ」というフワッとした概念を、バイトレベル・ファイルレベルで理解するフェーズだよ。

- 学ぶこと: イメージレイヤーの実態と Digest (SHA256ハッシュ) の仕組み
- 学ぶこと: Manifest と Manifest List (マルチアーキテクチャの正体)
- 学ぶこと: Image TypeとArtifact Typeの違い
実践: 実際にコマンドラインでECR上のマニフェストを覗き見してみる

## 📍 Phase 3: 最強のビルドエンジン「BuildKit」を飼い慣らす
今回の謎挙動の主犯格（!?）であるBuildKitの内部構造を理解するよ。

- 学ぶこと: 従来のビルドと BuildKit (buildx) のアーキテクチャの違い
- 学ぶこと: ビルドキャッシュの仕組み（ローカルキャッシュとリモートキャッシュ）
- 学ぶこと: docker build と docker compose build の裏側で何が起きているか（コマンドの違いがどうBuildKitに影響するか）

## 📍 Phase 4: CIの謎解きと最終トラブルシューティング
ここまで学んだ知識を総動員して、実際に起きた「56年前のイメージ」と「ローカルのキャッシュ汚染」の謎を解明・解決するよ！

- 学ぶこと: Reproducible Builds（再現可能なビルド）とタイムスタンプの仕様
- 学ぶこと: GitHub Actions上での最適なキャッシュ戦略とイメージのPull/Pushの挙動
ゴール: なぜ問題が起きたのかを同僚に分かりやすく説明し、修正PRを出せる状態になる！
